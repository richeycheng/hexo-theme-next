5. Spring AOP

5.1 概念

  • JDK动态代理：其代理对象必须是某个接口的实现，它是通过在运行期间创建一个接口的实现类来完成对目标对象的代理。
  • CGLIB代理：实现原理类似于JDK动态代理，只是它在运行期间生成的代理对象是针对目标类扩展的子类。CGLIB是高效的代码生成包，底层是依靠ASM（开源的Java字节码编辑类库）操作字节码实现的，性能比JDK强。
  • proxy-target-class：当该属性为true时，目标类本身被代理而不是目标类的接口，CGLIB动态代理将被创建，设置方式如下。
  • expose-proxy：有时候目标对象内部的自我调用将无法实施切面中的增强，可通过设置expose-proxy=”true”解决。

  • Spring的依赖JDK动态代理的AOP实现其实也是用了Proxy和InvocationHandler这两个东西，在自定义InvocationHandler中需要重写3个函数。
    • 构造函数，将代理的对象传入。
    • invoke方法，此方法中实现了AOP增强的所有逻辑。
    • getProxy方法，此方法返回JDK代理。
  • CGLIB动态代理使用了Enhancer类和MethodInterceptor接口，需要重写MethodInterceptor接口的intercept方法。

5.2 注意

  • 如果目标对象实现了接口，默认情况下会采用JDK动态代理实现AOP，也可以强制使用CGLIB动态代理实现AOP。
  • 如果目标对象没有实现接口，必须采用CGLIB代理，Spring会自动在JDK和CGLIB动态代理之间切换。
  • JDK动态代理只能对实现了接口的类生成代理，而不能针对类。
  • CGLIB是针对类实现代理，主要是对指定的类生成一个子类，覆盖其中的方法，因为是继承，所以该类或方法不要声明成final。

5.3 解析AOP自定义标签的关键点

  1. 通过parseCustomElement()解析AOP自定义标签aspecj-autoproxy时，会获取该标签对应的命名空间处理器AopNamespaceHandler，并调用其init()方法。
  2. 处理器AopNamespaceHandler类继承NamespaceHandlerSupport类，在init()方法中注册解析aspectj-autoproxy的解析器AspectJAutoProxyBeanDefinitionParser类。
  3. AspectJAutoProxyBeanDefinitionParser的parse()方法注册了AnnotationAwareAspectJAutoProxyCreator的BeanDefinition（包含proxy-target-class和expose-proxy属性）。
  4. AnnotationAwareAspectJAutoProxyCreator继承了AbstractAutoProxyCreator类，而AbstractAutoProxyCreator实现了BeanPostProcessor接口，当实例化bean时，会调用AnnotationAwareAspectJAutoProxyCreator继承自AbstractAutoProxyCreator类的postProcessAfterInitialization()方法。
